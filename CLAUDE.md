# Powder / Torov Visual — project rules

## ВАЖНО: порядок работы (read memory → then act)

Перед ЛЮБОЙ задачей в этом проекте:
1. **Сначала прочитать память** — индекс `C:\Users\mi\.claude\projects\C--powder-2025-01\memory\MEMORY.md`
   и релевантные файлы из него. Особенно `project-structure.md`, чтобы не путаться в пакетах.
2. **Потом действовать** — локализовать через Grep/Glob, прочитать узкий диапазон, внести минимальную правку.
3. **Собрать в основной jar** и проверить (см. ниже).
4. **Обновить память**, если узнал новый durable-факт (структура, гочи, билд).

Не начинать правки, не свериваясь со структурой проекта в памяти.

## Что это
Fabric-мод (MC 1.21.4, loom 1.9, Java 21). Два кодбейса: `powder.**` (ядро/фреймворк)
и `torovvisual.**` (портированный UI/HUD). Полная карта — в памяти `project-structure.md`.

## Сборка
- JDK 21 обязателен: `$env:JAVA_HOME="C:\Users\mi\.jdks\jdk-21.0.11+10"; .\gradlew.bat --no-daemon build --console=plain`
  (дефолтная java в PATH — 8, loom её отвергает). Детали — память `build-jdk`.
- Все изменения ДОЛЖНЫ попадать в `build\libs\torov-visual-1.0-1.21.4.jar` (юзер тестит только его).
- После правки `proguard.pro` собирать с `--rerun-tasks` (задача не отслеживает файл правил).

## Обфускация
Лёгкая, class-name-only (`-keepclassmembernames`), переименовываются только имена классов.
Member-rename ломает Мэйн меню (override-методы Screen). Подробно — память `obfuscation`.

- Минимальная дифф при багфиксе, без сайд-рефакторинга.
- Текстуры — только настоящий PNG (был баг JPEG-с-расширением-.png). См. `features-and-gotchas`.
- В чат — кратко (см. глобальный `~/.claude/CLAUDE.md`).
